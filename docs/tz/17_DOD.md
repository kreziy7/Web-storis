# 17 — Definition of Done (Чеклист готовности)

> [← Индекс](./00_INDEX.md)

---

## Аутентификация

- [ ] Регистрация работает, токены возвращаются корректно
- [ ] Логин работает, refresh token устанавливается в httpOnly cookie
- [ ] Access token автоматически обновляется через Axios interceptor
- [ ] Logout инвалидирует refresh token на сервере
- [ ] Валидация email/password работает (клиент + сервер)

---

## CRUD Напоминаний

- [ ] Create/Read/Update/Delete работают в online режиме
- [ ] Все операции мгновенны (optimistic UI без ожидания сервера)
- [ ] Валидация полей работает (title, dueDate, priority)
- [ ] Пагинация работает (`?page`, `?limit`)
- [ ] Фильтрация работает (`?isCompleted`, `?priority`)
- [ ] Сортировка работает (`?sortBy`, `?order`)

---

## Offline-first

- [ ] Приложение загружается без интернета (из IndexedDB)
- [ ] CRUD операции выполняются без интернета
- [ ] Операции добавляются в syncQueue при offline
- [ ] При восстановлении сети — автоматический sync запускается
- [ ] Конфликты разрешаются корректно (server wins по updatedAt)
- [ ] `pendingCount` отображается в UI
- [ ] Дедупликация операций в syncQueue работает
- [ ] Soft delete синхронизируется корректно

---

## PWA

- [ ] Lighthouse PWA score ≥ 90
- [ ] Приложение устанавливается через браузер
- [ ] Offline fallback страница показывается
- [ ] Service Worker кеширует статику при установке
- [ ] API запросы используют Network-First стратегию
- [ ] При обновлении SW — пользователь получает уведомление

---

## Push-уведомления

- [ ] Подписка сохраняется в MongoDB
- [ ] Уведомление приходит при приближении dueDate
- [ ] Клик по уведомлению открывает приложение
- [ ] Отмена подписки работает
- [ ] `410 Gone` от push service → подписка удаляется

---

## Безопасность

- [ ] `passwordHash` никогда не возвращается в API ответе
- [ ] Rate limiting работает (auth, sync, global)
- [ ] Input validation отклоняет невалидные данные
- [ ] Пользователь не видит чужие напоминания
- [ ] Единое сообщение об ошибке логина (no user enumeration)

---

## Качество кода

- [ ] TypeScript компилируется без ошибок (strict mode)
- [ ] Winston логирует все HTTP запросы
- [ ] Ошибки сервера логируются со stack trace
- [ ] `GET /health` возвращает 200
- [ ] Все async handlers обёрнуты в `asyncHandler`
- [ ] Нет console.log в production коде — только logger
